# ADR-0016: SBOM Generation and Publishing Strategy

**Status**: Accepted  
**Date**: 2026-04-07  
**Decision Makers**: Platform Team, CISO  

## Context

Software Bill of Materials (SBOM) is required for:

1. **Supply chain transparency** — knowing which third-party components are in the production artifact
2. **Vulnerability management** — correlating CVE feeds against the exact library inventory
3. **Compliance** — ISO 27001 Annex A control A.8.30 (Software Development Security) and emerging EU Cyber Resilience Act obligations
4. **Dependency management governance** — evidence that `pip-audit` and `npm audit` are operating against a known-good dependency graph

Prior to this decision, SBOM was generated as a CI artifact (ZIP download from GitHub Actions) but was not published to a stable, addressable location.

## Decision

We use **CycloneDX JSON** format generated by `cyclonedx-bom` as the primary SBOM artefact. The CI `sbom` job:

1. Generates `sbom.json` from the installed Python environment
2. Uploads it as a GitHub Actions artifact (retained 90 days) for per-build traceability
3. On every push to `main`, publishes the SBOM to the rolling **`sbom-latest`** GitHub Release using `gh release upload --clobber`

The `sbom-latest` release provides a stable URL for external consumers (security tooling, compliance auditors, dependency scanners).

## Consequences

**Positive:**
- Auditors can retrieve the current SBOM without GitHub Actions access
- Dependency scanners (Grype, Trivy, Dependabot) can be pointed at the stable release asset
- SBOM is versioned by commit SHA embedded in the release notes

**Negative:**
- The `sbom-latest` release is overwritten on every main push — historical SBOMs must be retrieved from GitHub Actions artifacts
- Requires `contents:write` permission on the `sbom` CI job

## Alternatives Considered

- GitHub Container Registry (OCI attestation via `cosign`): deferred — requires SLSA toolchain setup (backlog item EA-SBOM-SLSA)
- Per-release tagged SBOM: deferred — high storage growth; rolling latest is sufficient for current threat model
- No publishing (artifact-only): rejected — does not provide stable addressable URL for auditors

# Audit Trail Guide (D08)

**Owner**: Platform Engineering
**Last Updated**: 2026-04-03

---

## Overview

The Quality Governance Platform maintains comprehensive audit trails through structured request logging, database-level tracking, and evidence chain documentation.

---

## 1. Request-Level Audit Trail

Every API request is logged with the following fields by `src/infrastructure/middleware/request_logger.py`:

| Field | Description | Source |
|-------|-------------|--------|
| `request_id` | Unique correlation ID (UUID) | Generated per request or from `X-Request-ID` header |
| `method` | HTTP method (GET, POST, etc.) | Request |
| `path` | API endpoint path | Request |
| `status_code` | Response HTTP status | Response |
| `duration_ms` | Request processing time | Middleware timer |
| `user_id` | Authenticated user identifier | JWT token |
| `tenant_id` | Tenant context | JWT token / dependency resolution |
| `timestamp` | ISO 8601 timestamp | Server clock (UTC) |

### Log Storage

- **Staging**: Azure App Service log stream + Azure Log Analytics (90-day retention)
- **Production**: Azure App Service log stream + Azure Log Analytics (90-day retention)

## 2. Database-Level Tracking

| Mechanism | Implementation | Models |
|-----------|---------------|--------|
| `created_at` timestamp | Auto-set on insert | All domain models |
| `updated_at` timestamp | Auto-updated on modification | All domain models |
| `created_by` / `updated_by` | User ID from request context | Key domain models |
| Soft delete | `deleted_at` timestamp | Where applicable |
| Optimistic locking | `version` column | `InvestigationRun` (expanding to other models) |

## 3. Evidence Chain

For audit-related workflows, the platform maintains an evidence chain:

```
External Audit Import → ExternalAuditImportJob (source_filename, status, score_breakdown)
                           ↓ promote
                        AuditRun (reference_number, audit_date, findings)
                           ↓ linked
                        AuditFinding (finding_type, severity, clause_ids)
                           ↓ generates
                        CAPAAction (reference_number, status, linked finding)
                           ↓ linked
                        EnterpriseRisk (reference, likelihood, impact, risk_score)
```

Each entity carries:
- A unique `reference_number` generated by `ReferenceNumberService`
- Timestamps for creation and last modification
- Links to parent/child entities via foreign keys and JSON references
